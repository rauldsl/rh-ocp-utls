Sizing Formulas & Mathematical Auditing

Trace calculations step-by-step using the formulas documented directly within Red Hat OpenShift Virtualization Guide.

1. VM Memory Overhead Calculation

Every virtual machine requires a localized runtime overhead (in MiB) to run sidecar KVM processes and hypervisors.

Overhead per VM ≈ (0.002 * requested RAM MiB) + 218 MiB + (8 MiB * vCPUs) + (16 MiB * Graphics)
Interactive Tracing:
{{ vm.name }} ({{ vm.vcpu }}vC / {{ vm.memory }}GiB): (0.002 * {{ vm.memory * 1024 }}) + 218 + (8 * {{ vm.vcpu }}) + (16 * {{ vm.graphics }}) = {{ ((0.002 * (vm.memory * 1024)) + 218 + (8 * vm.vcpu) + (16 * vm.graphics)).toFixed(1) }} MiB

2. SMT / Hyperthreading core mapping

A physical CPU core exposes multiple execution threads, but 2 SMT threads do not provide 2x execution power. We calculate SMT effective capability by boosting thread difference.

Effective Cores = Physical Cores + ((SMT Threads - Physical Cores) * Thread Boost) [Bypassed if SMT OFF]
Active Sizing Calculation:

{{ cores }} + (({{ threads }} - {{ cores }}) * {{ htMultiplier }}) = {{ nodeSpecs.effectiveCores.toFixed(1) }} Cores {{ cores }} Cores (SMT turned OFF, physical boundary limit)

3. Kube-reserved Overhead Formulas

Red Hat dynamic allocations auto-reserve resources relative to effective resources for kube-reserved/system operations.

CPU reservation
Reserved CPU = 0.06 + (0.012 * (Operating Cores - 1)) [Min: 0.5C]
Active: Max(0.5, 0.06 + (0.012 * {{ (nodeSpecs.coresAfterHypervisor - 1).toFixed(1) }})) + 2.0 (OCV fixed) = {{ (nodeSpecs.kubeReservedCpu + 2.0).toFixed(2) }} Cores
Memory Reservation
First 4 GiB: 25% | Next 4 GiB: 20% | Next 8 GiB: 10% | Next 112 GiB: 6% | Remainder: 2% + 0.35 GiB OCV fixed + 4G Host
Active Allocation Reserved = {{ (nodeSpecs.kubeReservedRam + 4.35).toFixed(2) }} GiB

4. Dynamic Storage Overhead Allocation

If running converged storage like ODF/Ceph, local Storage Daemon processes (OSDs) require dedicated CPU & memory overheads to guarantee filesystem responsiveness.

ODF/Ceph formula:
Base Reservation: 2 Cores, 24 GiB RAM
Per NVMe SSD: +0.5 Cores, 4 GiB RAM
Per SAS SSD: +0.25 Cores, 2.5 GiB RAM
Per HDD: +0.20 Cores, 1.5 GiB RAM

Type: {{ storageType.toUpperCase() }} ({{ localDiskCount }} x {{ localDiskType.toUpperCase() }})
Reservation: {{ storageOverhead.coresPerNode }} Cores / {{ storageOverhead.ramPerNode }} GiB per node.
