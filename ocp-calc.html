Sizing Formulas & Mathematical Auditing

Trace calculations step-by-step using formulas from the official Red Hat OpenShift Virtualization Guide.

1. VM Memory Overhead Calculation

Every VM requires runtime overhead (MiB) for KVM sidecar processes.

Overhead per VM ≈ (0.002 × RAM MiB) + 218 MiB + (8 MiB × vCPUs) + (16 MiB × Graphics)
Interactive Tracing:
{{ vm.name }} ({{ vm.vcpu }}vC / {{ vm.memory }}GiB): (0.002×{{ vm.memory*1024 }}) + 218 + (8×{{ vm.vcpu }}) + (16×{{ vm.graphics }}) = {{ ((0.002*(vm.memory*1024))+218+(8*vm.vcpu)+(16*vm.graphics)).toFixed(1) }} MiB

2. SMT / Hyperthreading Core Mapping

Two SMT threads don't equal 2× execution power — we boost by thread multiplier.

Effective Cores = Physical Cores + ((SMT Threads − Physical Cores) × Thread Boost)
Active Calculation:
{{ cores }} + (({{ threads }} − {{ cores }}) × {{ htMultiplier }}) = {{ nodeSpecs.effectiveCores.toFixed(1) }} Cores {{ cores }} Cores (SMT OFF — physical boundary)

3. Kube-reserved Overhead Formulas

Red Hat auto-reserves resources proportionally for kube-reserved/system operations.

CPU Reservation
Reserved CPU = 0.06 + (0.012 × (Cores − 1)) [Min: 0.5C]
Active: Max(0.5, 0.06 + 0.012×{{ (nodeSpecs.coresAfterHypervisor-1).toFixed(1) }}) + 2.0(OCV) = {{ (nodeSpecs.kubeReservedCpu+2.0).toFixed(2) }} Cores
Memory Reservation
4G: 25% | +4G: 20% | +8G: 10% | +112G: 6% | Rest: 2%
Active Reserved = {{ (nodeSpecs.kubeReservedRam+4.35).toFixed(2) }} GiB

ODF/Ceph OSD daemons require dedicated CPU & memory per node.
